使用github.io和hugo建立博客

 2023-09-06 
目录
    • hugo
    • beautiful-hugo

github.io是Github免费提供的Pages服务，可发布静态页面，建立个人博客。

新建名为USERNAME.github.io的Repository，Settings->Pages->Deploy from a branch，选择分支和路径，访问https://USERNAME.github.io/即可。

hugo

hugo

下载hugo-extend版本。

新建：

1
2
3
4
5
6

hugo new site quickstart
cd quickstart
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo "theme = 'ananke'" >> hugo.toml
hugo server

添加文章：

1
2
3

hugo new content posts/my-first-post.md
hugo server -D    # with draft
hugo -D           # 生成静态页面，在public目录

其中：

  • new content在content子目录下创建md文件，默认draft为true，即草稿，默认不显示。文章放到post目录，否则不显示。
  • front-matter文件头按上面的命令会自动添加，或手动按格式编写
  • 任何子命令下可以通过-h查看help
  • 使用本地图片：在static目录下新建asserts目录，md中以![](/asserts/***.png)的方式引用
  • <!--more-->标签之前的内容会在文章列表中显示，可用来精简列表显示内容

beautiful-hugo

beautiful-hugo

创建名为blog_by_hugo的仓库，可设为私有仓库。创建名为USERNAME.github.io的仓库，设为公开仓库，存放静态页面。

1
2
3
4
5

git clone git@github.com:USERNAME/blog_by_hugo.git
hugo new site blog_by_hugo --force
cd blog_by_hugo
git submodule add https://github.com/halogenica/beautifulhugo.git themes/beautifulhugo
cp -r themes/beautifulhugo/exampleSite/* . -iv

可删掉默认的hugo.toml。

@echo off

set arg=%1
set blog=public
set username=sulpc

if "%arg%"=="prepare" (
    if exist %blog% (
        echo %blog% exsit, delete it
        rm -rf %blog%
    ) else (
        echo %blog% not exsited
    )
    git submodule init
    git submodule update
    git clone git@github.com:%username%/%username%.github.io.git %blog%
) else if "%arg%"=="commit" (
    cd %blog%
    rm -rf *
    cd ..

    hugo -D
    cd %blog%
    git add .
    git commit -m x
    git push
    cd ..

    git add .
    git commit -m x
    git push
) else (
    echo usage: ctrl.bat prepare/commit
)

从git上拉取到本地后，先执行ctrl.bat prepare，将public目录与USERNAME.github.io关联。

本地添加完文章，测试无误后，执行ctrl.bat commit将修改推入git仓库。
